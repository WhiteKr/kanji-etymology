## 완료 기준

- **P2-1**: 시스템 설정(prefers-color-scheme) 자동 추종 + 수동 토글(localStorage 저장).
  라이트/다크 양쪽에서 전 페이지 가독성 확인.
- **P2-2**: 부수 8개 본문이 한자 페이지 수준의 어원 서술(자형 기원, 부수로서의 역할,
  소속 한자와의 연결)을 갖춤. build-data 검증 통과.
- **P2-3**: 검색 입력 중 후보 목록이 실시간 갱신, 키보드(↑↓/Enter/Esc) 탐색 가능.
  기존 검색 테스트 전체 통과 + 신규 테스트.
- **P2-4**: Tanos N5 한자 목록 전체가 content/kanji/에 존재. 신규 글자가 요구하는
  부수 파일 추가 포함. build-data 검증(참조 무결성 포함) 통과. 검수 노트에 신규분
  자신도 낮은 항목 기록.
- **P2-5**: KANJIDIC2 XML을 파싱해 초안 frontmatter 스켈레톤을 생성하는 서브커맨드.
  픽스처 기반 테스트 통과. (실데이터 다운로드는 라이선스 고지 포함 별도 실행 —
  CC-BY-SA, edrdg.org)

## 진행 순서

1. **Wave 1 (병렬)**: P2-1 다크 모드 / P2-2 부수 깊이 / N5 갭 분석(P2-4 사전 작업)
2. **Wave 2 (병렬)**: P2-3 자동완성 / P2-4 신규 한자 배치 작성
3. **Wave 3**: P2-5 임포트 도구 → 전체 검증 → 배포 확인

각 마일스톤 완료 시 커밋 (자동 재배포). 콘텐츠 신규분은 검수 노트에 2차 검수
대상으로 축적한다.
